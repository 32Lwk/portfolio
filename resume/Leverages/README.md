## LaTeX で PDF をビルドする場合

XeLaTeX で PDF を生成します（日本語フォント指定のため）。

```bash
cd resume/Leverages
xelatex -jobname=Rirekisho_Yuto_Kawashima rikurekisho.tex
```

またはプロジェクトルートから:

```bash
xelatex -jobname=Rirekisho_Yuto_Kawashima -output-directory=resume/Leverages resume/Leverages/rikurekisho.tex
```

生成される PDF: `Rirekisho_Yuto_Kawashima.pdf`

## LaTeX と履歴書について

日本式履歴書は「罫線で区切られた枠」が多く、LaTeX の表（tabular + multirow）で再現すると次のような不具合が出やすいです。

- **日付欄**: 「年・月・日」を別セルにすると「2026年2月19日 年 月 日」のように二重表示になる → 日付は1セルにまとめて表示する形にしています。
- **写真＋複数行**: 写真セルと氏名の複数行をまたぐセルで、行の高さが合わずレイアウトが崩れる／Overfull が出る → 写真を `\parbox` で高さ固定し、表の列数を 3 列に整理しています。
- **長い住所**: 1行で書くとセルからはみ出す → 内容列を `p{...}` にして折り返し、必要なら `\parbox` で幅を指定しています。

修正後も Overfull 警告が少し出る場合は、写真サイズをさらに小さくするか、余白（geometry の top）を調整してください。  
どうしても LaTeX で収まりが悪い場合は、Word や Google ドキュメントの履歴書テンプレートに内容を写して PDF 化する方法も有効です。

## 強み・レバレジーズへの思い・Live Demo・ポートフォリオ・GitHub の共有のしかた

- **履歴書の「特技・趣味・自己PR」欄**  
  強み・レバレジーズへの思いを本文に織り込み、**文末で「作品・開発の詳細は下記で公開しています」と書き、その下にポートフォリオ・GitHub・医薬品相談ツール Demo の URL を記載**すると、1枚で伝わります。HTML ひな形・LaTeX ともこの形で反映済みです。
- **志望動機の最後に1行添える**  
  「詳細はポートフォリオ（yutok.dev）・GitHub で公開しています」と書くと、面接官がリンクを探しやすくなります。
- **提出時に別紙を添付する**  
  英文レジュメ（`Resume_Yuto_Kawashima.pdf`）に Live Demo・GitHub・portfolio のリンクが載っているため、履歴書と一緒に「職務経歴書・作品集として」添付すると、技術面のアピールが伝わりやすくなります。
- **面接で口頭で伝える**  
  「医薬品相談ツールはブラウザでデモ可能です。お時間あればポートフォリオのURLからご覧ください」と一言添えると、その場で開いてもらいやすくなります。
