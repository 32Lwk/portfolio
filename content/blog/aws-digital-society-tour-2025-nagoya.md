# AWS デジタル社会実現ツアー 2025 名古屋に参加して

2025年8月19日、AWSジャパン主催の「AWS デジタル社会実現ツアー 2025 名古屋」に参加し、開発中のチャット型医薬品相談ツールを発表する機会をいただきました。この記事では、参加の経緯、発表内容、そして得られた学びについて振り返ります。

## 参加の経緯

AWS デジタル社会実現ツアーは、AWSを活用した社会課題解決の取り組みを発表するイベントです。名古屋会場では、地域の学生やエンジニアが集まり、それぞれのプロジェクトを紹介しました。

私がこのイベントに参加した理由は、開発中のチャット型医薬品相談ツールが「デジタル社会の実現」というテーマに合致していると考えたからです。特に、以下の点が評価されました：

- **社会課題の解決**: ドラッグストアでの現場経験から生まれた、高齢者や外国人を中心とした医薬品アクセスの課題解決
- **クラウド技術の活用**: GCP Cloud Runへの移行経験を含む、クラウドインフラの実践的な運用
- **AI技術の適切な活用**: LLMとルールベースのハイブリッド設計による、安全性を重視したAI実装

## 発表内容

### プロジェクト概要

「大規模言語モデルを用いた安全で最適な市販薬提案チャットツール」として、以下の点を中心に発表しました：

1. **開発の背景**
   - ドラッグストアでの現場経験から見えた課題
   - 過疎地での医薬品アクセスの困難さ
   - ネット購入の普及による相談機会の減少

2. **技術的な特徴**
   - ハイブリッド推奨システム（ルールベース＋LLM）
   - 安全性を最優先にした設計思想
   - 多言語対応・アクセシビリティ対応

3. **クラウドインフラの運用**
   - RenderからGCP Cloud Runへの移行
   - Cloud SQLからNeon PostgreSQLへの移行
   - サーバーレスアーキテクチャの選定理由

### 反応とフィードバック

発表後、多くの参加者から質問やフィードバックをいただきました。特に印象的だったのは：

- **安全性への関心**: 「医療分野でAIを使う際の安全性をどう担保しているか」という質問が多く、この分野への関心の高さを実感しました
- **実用性への評価**: 実際の現場経験に基づいた開発である点が評価され、理論だけでなく実践的な価値があると感じてもらえました
- **技術的な深掘り**: ハイブリッドシステムの設計思想や、クラウド移行の判断基準について、より詳しい説明を求められる場面もありました

## 学びと気づき

### 1. 社会課題の明確化の重要性

他の参加者の発表を聞いて感じたのは、**社会課題を明確に定義することの重要性**です。技術的な実装だけでなく、「なぜこの課題を解決する必要があるのか」「誰が困っているのか」を具体的に示すことで、プロジェクトの価値がより伝わりやすくなります。

私の発表でも、ドラッグストアでの具体的なエピソードや、過疎地での医薬品アクセスの課題を紹介したことで、多くの方に共感していただけたと感じています。

### 2. 技術と社会課題の接続

AWSのようなクラウド技術は、単なる技術的なツールではなく、**社会課題を解決するための手段**であることを再認識しました。クラウドインフラの選定や運用においても、「なぜこの技術を選んだのか」「どのように社会課題の解決に貢献するのか」を常に意識することが重要です。

### 3. コミュニティとのつながり

イベントを通じて、名古屋地域のエンジニアや学生の方々とつながることができました。同じ地域で活動する仲間がいることで、モチベーションも高まり、今後の開発にも良い影響を与えてくれています。

## 開発者としての想い

このイベントに参加して、改めて感じたのは、**「システムを誤らせない設計」の重要性を多くの方に理解していただけた**ことです。

医療分野でAIを活用する際には、技術的な高度さだけでなく、安全性と正確性が何よりも重要です。LLMに判断を丸投げするのではなく、ルールベースの制約を組み合わせることで、再現性と安全性を担保する設計を説明した際、多くの方が「なるほど」と納得してくださったのが印象的でした。

また、クラウド移行の経験についても、単なる技術的な移行ではなく、「なぜ移行が必要だったのか」「どのような判断基準で選定したのか」を説明することで、より深い理解を得ていただけたと感じています。

## 今後の展望

このイベントでの経験を踏まえ、今後も以下の点を意識して開発を進めていきたいと考えています：

1. **社会課題の明確化**: 技術的な実装だけでなく、解決したい課題を常に明確に示す
2. **安全性の追求**: 医療分野でのAI活用において、安全性を最優先にした設計を継続する
3. **コミュニティとの交流**: 地域のエンジニアや学生とのつながりを大切にし、互いに学び合える環境を作る

AWS デジタル社会実現ツアーでの経験は、私にとって非常に貴重な学びの機会となりました。今後も、社会課題の解決に貢献できるエンジニアとして、技術と社会の接続を意識しながら開発を続けていきたいと思います。
